Release Notes for Touch Charts 1.0.0

Release Notes: Release Notes: August 9, 2011
Version Number: 1.0.0

  • General Notes
    • This release is shipping with the current (1.1.1) head branch of Sencha Touch. Version 1.1.1+ of Sencha Touch fixes several issues with MultiTouch detection and orientation problems on non-iOS devices. Touch Charts will work with 1.1.0, but does not support Android Tablets on that release.
  • Fixes / Changes since last release
    • Added :highlight to allow styling of the highlights via SASS.
    • Added ability to start with a default zoom (startZoom) and pan (startPan) values for the PanZoom interaction.
    • Defered drag event initialization so that a chart without drag interactions can be used within a Carousel.
    • Added Linux versions of Fashion.
    • Added Pan/Zoom interaction to the line example.
    • Stock example now uses a time axis.
    • Removed custom theme from the Column example.
    • Docked legends now hide if swiped in the direction they came from (side-docked only).
    • Reformatted some tab/spaces in the Line example and properly case the series titles.
    • Phone chart controls now use regular buttons with masks and can be styled via SASS.
    • Legend style sheets are much simpler and have a shadow.
    • Legend sheet animations are now faster.
    • Fix lineWidth in the Canvas engine.
    • Fix highlight configuration order.
  • Known Issues
    • Mixins for Touch 1.1.0 are not currently being detected properly in the documentation application and are not displaying in their main class.

Release Notes for Touch Charts 1.0.0.rc

Release Notes: Release Notes: August 2, 2011
Version Number: 1.0.0.rc

  • General Notes
    • This release is shipping with the current (1.1.1) head branch of Sencha Touch. Version 1.1.1+ of Sencha Touch fixes several issues with MultiTouch detection and orientation problems on non-iOS devices. Touch Charts will work with 1.1.0, but does not support Android Tablets on that release.
  • Fixes / Changes since last release
    • Changed PieRotate interaction to Rotate, and make it apply to radar series/radial axes as well as pies.
    • Allow interactions with no custom config to be specified with just their type as a String value rather than having to wrap in an Object. For example interactions: ['rotate'] is now valid.
    • Radial Axes are now a bit thicker and display better on devices.
    • Circles now display properly on Blackberry devices.
    • Loading mask in the Energy app now displays the throbber properly.
    • The help icon now displays properly in the Column example.
    • Fix Radar styling issues.
    • Added a workaround for iOS limitation on the size of a canvas element (a bit less that 4 million pixels). You can now zoom very deep into charts without losing the display.
    • Streamlined the Combo example.
    • You can now split joined legend items by double-tapping on them.
    • Fix issues with the Win32 Fashion executable.
    • A marker now always displays in the location of a highlight.
    • Pie labels will now be hidden if they do not fit within the pie slice. This behavior can be changed to use a callout label as well.
    • Pie callouts can now be styled properly.
    • Added a loadmask to be displayed on tabswitch in the Energy App.
    • Fixed the initial slider position in the Energy App.
    • Axes no longer clip when they aren't in a zoomed state.
    • Legends are no longer scrollable unless necessary.
    • Prevented unnecessary redraws during layout operations.
    • Fix animation issue which was occasionally causing a 1 degree gap in Pie charts.
    • When tap-holding legend items, prevent items from becoming draggable which do not have another matching item with which they can be combined.
    • Fix image sprites.
    • Add new Shape 'image' which allows images to be used as markers.
    • Guides have been updated to reflect Touch Charts specifically.
    • Added info and comparison interactions to the WorldData app
  • Known Issues
    • Mixins for Touch 1.1.0 are not currently being detected properly in the documentation application and are not displaying in their main class.
    • Linux versions of Fashion to be added

Release Notes for Touch Charts 1.0.0.beta

Release Notes: Release Notes: July 21, 2011
Version Number: 1.0.0.beta

Initial Public Release

  • General Notes
    • This release is shipping with the current (1.1.1) head branch of Sencha Touch. Version 1.1.1+ of Sencha Touch fixes several issues with MultiTouch detection and orientation problems on non-iOS devices. Touch Charts will work with 1.1.0, but does not support Android Tablets on that release.
  • Known Issues
    • Radar Axis does not display properly on BlackBerry
    • Mixins for Touch 1.1.0 are not currently being detected properly in the documentation application and are not displaying in their main class.
    • Zooming too far into a chart on an iPad causes a display issue
    • Styling of the Radar Chart in the Energy App is not correct
  • Forthcoming
    • Default gestures for interactions are still under review and may change before release
    • SASS syntax is still under review and may change before release
    • Additional markers will be added during an ItemInfo interaction to better highlight what was selected
    • Collision detection on the pie will hide/move text when it cannot fit within a slice
    • Callout examples are to be added back in
    • Styling Guide to be added to the documentation
    • Linux versions of Fashion to be added
    • Spin a radar chart
    • Double-tap on a grouped legend item will ungroup the items
